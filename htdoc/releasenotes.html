Stage Reservation App V0.1

Release Notes

Introduction
Welcome to the V0.1 release of Compliance Stage Reservation Manager App (CSRM). CSRM aims to improve and optimize stage reservation processes and aid developers by increasing usability and providing a friendly UI to facilitate stage management. CSRM automates the reservation process and prevents accidental overwriting of reservations, ensuring users are informed when overwriting reservations and reminding them to inform appropriate spec leads/ dl of changes in stage reservation.

How to use the App?
  • Navigate to the link : http://10.238.85.62/CSRM/index.html
  • Enter stage reservation details in the Reservation form located above the table.
    • Choose the stage from the drop down menu
    • Select the period for which you want to reserve the stage for(the To date must be greater than the From date)
    • Enter the Spec Id and Spec Name of the Spec for which the stage is being reserved.You can choose to fill either of the fields as the other field is auto filled as soon as one of them is entered.
  • You can reserve stages only for specs whose details are present in the database. If you wish to reserve a stage for new spec then proceed with the following steps :
    • Click on the Add New Spec link which can be found at the top left corner of the home page.
    • You will be redirected to a page where you can enter the details of the new spec that is to be added
    • Fill in the details and click Submit. (While Spec ID and DL-email values may be left blank , the Spec Name and the Spec lead email are mandatory fields. If their values are not entered , you will not be able to add the new spec)
    • Once the spec has been added , you can proceed with reserving a stage for that spec.
  • Once a reservation has been made , you can view it in the table by choosing the correct year and month in the monthpicker which is at the head of the table.

Capabilities of V0.1
  • Prevents accidental Overwriting of reservations
    • While reserving a stage , if the application detects that you are trying to overwrite a reservation made by another spec for the same stage, a warning is issued.
    • You can choose to cancel the current operation or can continue with the overwriting
    • In case you wish to overwrite , you will be notified of the details of the spec lead of the overwritten spec so that you can contact them about the changes in stage reservation.
  • User friendly interface
    • This application makes use of jqgrid - an external js library which comes with various features:
      • View records by year
      • View records by month
      • Choose to display only certain columns(for example , you may wish to view reservations only for a specific stage)
    • All the above functionalities can be done by clicking on the choose column button at the bottom left corner of the table.
  • Enhanced search function
    • Click on the find records option (search icon) located at the bottom left corner of the table , this opens up a widget which allows you to search the records with filters. like
      • Search by Columns
      • Add qualifiers: equal, less than, greater...
      • Multiple search criteria
      • Match any/all rules

Limitations
  • V0.1 is not compatible with Internet Explorer version 8 and below.
  • The sorting functionality in the table does not work in Chrome
  • Table width may not be fully expanded in Chrome
